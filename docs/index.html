Quartz Network Library

Quartz is an open source DX-oriented network library for Roblox that provides type safety, rate limiting, and advanced networking features with a clean, intuitive API.

Why Quartz?

  • Type Safety - Built-in validation for all Roblox data types
  • Performance - Optimized for high-frequency networking
  • Security - Configurable rate limiting and validation
  • Developer Experience - Clean API with comprehensive documentation
  • Flexibility - Support for both reliable and unreliable networking

Basic Setup

-- Server Script
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Quartz = require(ReplicatedStorage.Quartz)
local Server = Quartz.Server()

-- Client Script (LocalScript)
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Quartz = require(ReplicatedStorage.Quartz)
local Client = Quartz.Client()

Quick Example

-- Server: Create typed event
local PlayerDataEvent = Server.Event("PlayerData")
    .Expects(Quartz.string, Quartz.number, Quartz.boolean)

PlayerDataEvent.OnFire(function(player, name, level, isPremium)
    print(`Player {name} level: {level} premium: {isPremium}`)
end)

-- Client: Fire event
local PlayerDataEvent = Client.Event("PlayerData")
PlayerDataEvent.Fire("John", 25, true)

Server API

Events

server.Event(name, unreliable, mode, ...validators)
Creates a new server event. Returns an event object with methods for handling and firing events.

Event Methods

event:OnFire(callback)
Sets the callback function when the event is fired from the client.
event:Fire(player, ...)
Fires the event to a specific player.
event:Expects(...types)
Sets validation rules for event parameters.

Functions

server.Function(name)
Creates a new remote function. Returns a function object for handling remote invocations.

Function Methods

function:OnInvoke(callback)
Sets the callback for when the function is invoked from the client.
function:Invoke(player, ...)
Invokes the remote function on a specific client.

Client API

Events

client.Event(name, unreliable, ...validators)
Creates or retrieves a client event. Returns an event object for client-server communication.

Event Methods

event:OnFire(callback)
Sets callback for when server fires this event to the client.
event:Fire(...)
Fires the event to the server.

Functions

client.Function(name)
Creates or retrieves a client function for invoking server methods.

Function Methods

function:Invoke(...)
Invokes the function on the server and returns the result.

Advanced Validators

-- Range validation
Quartz.range(min, max)  -- Number must be between min and max

-- Value constraints
Quartz.min(value)       -- Number must be at least value
Quartz.max(value)       -- Number must be at most value

-- Custom validation
Quartz.custom(function(value)
    return type(value) == "string" and #value <= 50
end)

Examples

Basic Chat System

-- Server
local ChatMessage = Server.Event("ChatMessage")
    .Expects(Quartz.string)
    .WithRateLimit(20, 10) -- 20 messages per 10 seconds

ChatMessage.OnFire(function(player, message)
    if #message > 100 then return end
    
    -- Broadcast to all players
    ChatMessage.FireToMultiple(game.Players:GetPlayers(), player.Name, message)
end)

-- Client
local ChatMessage = Client.Event("ChatMessage")

-- Send message
ChatMessage.Fire("Hello world!")

-- Receive messages
ChatMessage.OnFire(function(playerName, message)
    print(`{playerName}: {message}`)
end)

Player Data Management

-- Server: Data retrieval function
local GetPlayerData = Server.Function("GetPlayerData")
    .OnInvoke(function(player, dataKey)
        return PlayerData[player][dataKey]
    end)

-- Client: Request data
local GetPlayerData = Client.Function("GetPlayerData")
local playerCoins = GetPlayerData.Invoke("coins")
